## Summary

GPDStore blog review of the BIWIN Mini SSD 1517, a new compact storage format tested in the [[gpd-win-5]].

## Key Specs

| Spec | Value |
|------|-------|
| Interface | PCIe Gen4x2, NVMe 1.4 |
| Flash type | 3D TLC |
| Capacities | 512 GB, 1 TB, 2 TB |
| Sequential read | Up to 3700 MB/s |
| Sequential write | Up to 3400 MB/s |
| Random read 4K | Up to 550K IOPS |
| Random write 4K | Up to 650K IOPS |
| DRAM | DRAM-less |
| Max power | 2.3 W (read/write) |
| Idle power | 60 mW (6 mW low power state) |
| Dimensions | 15 x 17 x 1.4 mm |
| Endurance | Up to 1500 TBW |
| MTBF | 1,500,000 hours |

## Benchmark Results (GPD WIN 5 — PCIe 4x1 bottleneck)

- **CrystalDiskMark:** 1785 MB/s read, 1744 MB/s write
- **AS SSD:** Read 1470, Write 1632, Overall 3855
- **ATTO:** From 44/41 MB/s (512 B) to 1.66/1.63 GB/s (64 MB files)

Note: GPD WIN 5 uses PCIe 4x1 (max ~2000 MB/s). Full PCIe 4x2 would achieve 3700+ MB/s.

## Device Compatibility

Currently confirmed: [[gpd-win-5]], ONEXPLAYER Super X (announced).

## Verdict

Revolutionary compact storage. Massive leap over MicroSD (~100 MB/s). Bottlenecked by WIN 5's 4x1 interface — full potential awaits PCIe 4x2 devices.
